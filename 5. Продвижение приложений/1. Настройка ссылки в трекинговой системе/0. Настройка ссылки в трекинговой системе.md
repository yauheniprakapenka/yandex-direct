# Настройка ссылки в трекинговой системе

Получить данные о новых установках из магазина приложений можно с помощью трекинговой системы. Она сопоставляет установки и действия в приложении с переходами по объявлению и передает эти данные в Директ для подсчета конверсии.

Подключение трекинговой системы дает возможность:

- [оптимизировать](ru/strategies/average-cpa-mobile-apps) кампанию на установки и события в приложении;
- точно собирать статистику по установкам и событиям внутри приложения, которая будет доступна и в Директе, и в трекинговой системе.

Директ поддерживает следующие трекинговые системы: [AppMetrica](https://appmetrika.yandex.ru), [AppsFlyer](http://www.appsflyer.com/), [Adjust](https://www.adjust.com/), [Branch](https://branch.io/), [Kochava](http://www.kochava.com/), [MyTracker](https://tracker.my.com/), [Singular](https://www.singular.net/), [Tenjin](https://www.tenjin.com/) и [AirBridge](https://www.airbridge.io/). Поддержка SKAdNetwork для кампании на iOS 14.5 работает в системах AppMetrica, AppsFlyer и Adjust.

После запуска кампании можно [сменить](ru/products-mobile-apps-ads/change-tracking-system) трекинговую систему.

## Верификация приложения в трекинговой системе

Пройдите верификацию приложения, чтобы:

- настраивать ретаргетинг на пользователей, которые достигли целей, указанных в трекере;
- получать статистику по технологии SKAdNetwork для кампаний на iOS 14.5 и выше.

##### AppMetrica

Если AppMetrica и Директ находятся на разных логинах, для верификации приложения в кабинете AppMetrica необходимо выдать доступ логину Директа. Если логины совпадают, верификация будет пройдена автоматически.

##### Adjust или AppsFlyer

Укажите [App ID](#add-aps) при настройке интеграции, и приложение будет верифицировано после получения первого постбека от мобильного трекера (вы увидите значок ![Верификация](https://cdn-viewer.diplodoc.com/docs-assets/support-direct-pro/rev/r20128576/ru/_assets/button-ok-new.png) в настройках приложения).

В случае с AppsFlyer, чтобы верификация прошла успешно и Директ получал обогащенные данные о конверсиях, в настройках интеграции в трекере с Директом отключите [фреймворк агрегированной расширенной конфиденциальности](https://support.appsflyer.com/hc/ru/articles/360018515798-Фреймворк-Агрегированной-расширенной-конфиденциальности).

## Добавьте приложение в библиотеку Директа

Чтобы настроить связь Директа и трекинговой системы, добавьте приложение в библиотеку.

1. В меню слева нажмите **Библиотека** и выберите **Мобильные приложения**. Нажмите **Добавить приложение**.
2. Укажите ссылку на приложение в AppStore или Google Play. Директ загрузит из магазина иконку, цену, рейтинг, количество оценок и технические характеристики приложения. Используйте App ID, полученный после добавления приложения, для настроек [трекинговых ссылок](#tracking-links).

   ![App ID](https://cdn-viewer.diplodoc.com/docs-assets/support-direct-pro/rev/r20128576/ru/_assets/app-id-img.png)

   Обратите внимание, система получает данные для той страны, которая указана в ссылке на магазин приложений. По умолчанию данные загружаются для России. Используйте специальные параметры, чтобы объявление вело на магазин приложений для определенной страны.

   **AppStore**

   Страна указывается в ссылке после `apps.apple.com`.

   ```
   https://apps.apple.com/ru/app/yandex-browser-for-iphone/id483693909
   ```

   **Google Play**

   Страна указывается в параметре `gl`.

   ```
   https://play.google.com/store/apps/details?id=com.yandex.browser&gl=ru
   ```

После настройки трекинговой ссылки, вернитесь в библиотеку приложений и укажите ее для нужного приложения. Создавать кампании для приложения можно только с трекинговыми ссылками, которые указаны в библиотеке приложений. Если вы хотите использовать разные трекинговые системы для одного приложения, например, вы работаете с приложением и в [AppMetrica](https://appmetrika.yandex.ru), и в [Appsflyer](http://www.appsflyer.com/), указать их вместе для одного приложения не получится. Поэтому добавьте приложение в библиотеку 2 раза и укажите в одном случае трекинговую ссылку из одного трекера, а в другом — из другого.

## Трекинговые ссылки

### Продвижение на мобильных устройствах и планшетах

Чтобы связать трекинговую систему и кампании в Директе, необходимо добавить трекинговую ссылку в объявления. При этом в коде мобильного приложения должен быть прописан SDK трекинговой системы: это программный модуль, который передает системе информацию об установках и запусках.

В ссылке можно передавать различные параметры и метки, в том числе UTM-метки:

```
https://redirect.appmetrica.yandex.com/serve/7438548069196542965/?click_id={logid}&source={source}
```

С их помощью можно отслеживать различные статические показатели, например, источники трафика. Подробнее об [использовании параметров и меток](ru/statistics/url-tags).

> **Важно**
>
> Ссылка создается рекламодателем в трекере.

На вкладках ниже описано, как создать трекинговые ссылки в поддерживаемых Директом трекинговых системах.

> **Внимание**
>
> Если вы используете другую трекинговую систему, вы можете обратиться к ее владельцу (разработчику) с просьбой об интеграции с Директом. Заявки на подключение трекинговых систем к Директу принимаются только от владельцев этих систем через [форму обратной связи](ru/troubleshooting/tracking-system). Взаимодействие Директа с трекинговыми системами подробно описано в [документации](https://yastatic.net/s3/doc-binary/src/support/direct/ru/yandex-mobileapp-postback-guide.pdf).

#### AppMetrica

Чтобы получить трекинговую ссылку, выполните следующие шаги:

1. Перейдите на вкладку **Трекеры** и нажмите **+Создать трекер**.
2. Выберите приложение в выпадающем списке **Приложения**. SDК уже должен быть прописан в коде приложения. Подробнее о том, как добавить SDK для [Android](https://appmetrica.yandex.ru/docs/mobile-sdk-dg/android/about/android-initialize.html) и [iOS](https://appmetrica.yandex.ru/docs/mobile-sdk-dg/ios/ios-quickstart.html).
3. Выберите Yandex.Direct в выпадающем списке **Партнер**.
4. Скопируйте созданную системой трекинговую ссылку.

   > **Важно**
   >
   > Ссылка должна содержать обязательный параметр `click_id={logid}`.
   >
   > При создании ссылки часть параметров добавляется автоматически. Используйте ссылку со всеми предзаполненными параметрами: они нужны для корректной работы атрибуции и включают стандартный набор для аналитики.

   Описание параметров ссылки см. в разделе [Параметры трекинг-ссылки](https://appmetrica.yandex.ru/docs/ru/mobile-tracking/tracking-specification) документации [AppMetrica](https://tech.yandex.ru/appmetrica/).

5. Вставьте полученную трекинговую ссылку в объявление.

Должна получиться ссылка вида:

```
https://redirect.appmetrica.yandex.com/serve/7438548069196542965?google_aid_sha1={GOOGLE_AID_LC_SH1}&ios_ifa_sha1={IDFA_LC_SH1}&android_id_sha1={ANDROID_ID_LC_SH1}&rnd=%aw_random%&device_type={device_type}&source_type={source_type}&source={source}&ios_ifa={IOSIFA}&device_ip={client_ip}&google_aid={GOOGLEAID}&click_id={LOGID}&search_term={keyword}&region_name={region_name}&phrase_id={phrase_id}&android_id={ANDROIDID}&position_type={position_type}&campaign_id={campaign_id}
```

Или app-to-app ссылка вида:

```
https://4321123.redirect.appmetrica.yandex.com/?appmetrica_tracking_id=7438548069196542965&referrer=reattribution%3D1&google_aid_sha1={GOOGLE_AID_LC_SH1}&ios_ifa_sha1={IDFA_LC_SH1}&android_id_sha1={ANDROID_ID_LC_SH1}&rnd=%aw_random%&device_type={device_type}&source_type={source_type}&source={source}&ios_ifa={IOSIFA}&device_ip={client_ip}&google_aid={GOOGLEAID}&click_id={LOGID}&search_term={keyword}&region_name={region_name}&phrase_id={phrase_id}&android_id={ANDROIDID}&position_type={position_type}&campaign_id={campaign_id}
```

Подробнее см. в разделе [Создание трекера](https://appmetrica.yandex.ru/docs/ru/mobile-tracking/yandex-direct-upc-tracking) документации [AppMetrica](https://tech.yandex.ru/appmetrica/).

> **Примечание**
>
> Для корректной работы стратегии [Максимум конверсий](ru/strategies/average-cpa-mobile-apps) убедитесь, что в трекере включена передача данных об установках приложения.

Настройки интеграции подробно описаны в [документации AppsFlyer](https://support.appsflyer.com/hc/en-us/articles/207033816-Integrated-partners-setup-guide#partner-setup).

Рекомендуемые настройки для наиболее эффективного продвижения в Директе:

- **Выберите партнера Yandex Direct**

  1. В интерфейсе AppsFlyer в меню выберите **Collaborate** → **Partner Marketplace**.

- **Настройте нужные параметры**

  1. Нажмите кнопку **Set up integration**.

     ![Настройка AppsFlyer](https://cdn-viewer.diplodoc.com/docs-assets/support-direct-pro/rev/r20128576/ru/_assets/AppsFlyer-setting.png)

  2. Активируйте опцию **Activate partner**.

  > **Совет**
  >
  > Для iOS-приложений настройку **Advanced Privacy** оставьте выключенной. Это не повлияет на сбор данных и поможет улучшить обучение стратегий.

  3. Скопируйте **Yandex App ID** из библиотеки мобильных приложений. Где найти [App ID](#add-aps).
  4. Вставьте скопированный **App ID** в поле **Yandex App ID**.
  5. Включите опцию **In-app events postback** для передачи событий.
  6. Выберите события, которые вы хотите передавать в Директ. Их можно будет просматривать в Мастере Отчетов и использовать для оптимизации ставок.

  Для каждого передаваемого события укажите название, под которым оно будет показываться в Директе.

  Выберите способ отправки событий **All media sources, including organic**.

  Выберите **Values & revenue** для передачи выручки. Ее можно будет увидеть в Мастере Отчетов.

  Нажмите **Add Event** и добавьте **af_app_opened**, чтобы включить передачу сессий.

- **Создайте трекинговую ссылку в разделе на вкладке Attribution Link.**

  В трекинговых ссылках можно использовать [динамические параметры Директа](ru/statistics/url-tags#dynamic). Если вы планируете использовать атрибуцию на показ в своих кампаниях, убедитесь что она включена в настройках трекера: **Конфигурация** → **Интеграции** → **Атрибуция на показ**.

  Подробнее создание трекинговой ссылки описано в [помощи AppsFlyer](https://support.appsflyer.com/hc/en-us/articles/207033816-Integrated-Partner-Configuration). Если у вас возникли вопросы по работе ссылки, обратитесь в [службу поддержки AppsFlyer](https://support.appsflyer.com/hc/en-us/requests/new).

  > **Важно**
  >
  > Убедитесь, что используются обязательные параметры:
  >
  > - `pid=yandexdirect_int`;
  > - `clickid={logid}`;
  > - `&oaid={oaid}` — идентификатор устройства Android без сервисов Google;
  > - `&gps_adid={google_aid}` — идентификатор устройства Android;
  > - `&idfa={ios_ifa}` — идентификатор устройства iOS;
  > - `is_retargeting=true` — для [ретаргетинговых кампаний](ru/products-mobile-apps-ads/retargeting-mobile-apps);
  > - `&ya_click_id={logid}` — идентификатор клика/показа.
  >
  > Для корректной работы стратегии [Оптимизация конверсий](ru/strategies/average-cpa-mobile-apps) убедитесь, что в трекере включена передача данных об установках приложения.

  Рекомендуем включить передачу статуса органичности данных в трекере — количество передаваемых данных увеличится, а эффективность кампаний вырастет.

  Для передачи статуса перейдите в раздел **Active integrations**, выберите **Yandex Direct** и в блоке **General settings** включите опцию **Share organic/non-organic indication**.

  ![Share organic/non-organic](https://cdn-viewer.diplodoc.com/docs-assets/support-direct-pro/rev/r20128576/ru/_assets/appsflyer-share.jpeg)

#### Branch

Настройки интеграции подробно описаны в [документации Branch](https://help.branch.io/using-branch/docs/universal-ad-partner-integration-guide).

- **Выберите партнера Yandex.Direct**

  1. В интерфейсе Branch перейдите на страницу [Ads](https://dashboard.branch.io/ads).
  2. Выберите **Partner Management** → **Yandex.Direct**.

     ![Branch — Partner Management](https://cdn-viewer.diplodoc.com/docs-assets/support-direct-pro/rev/r20128576/ru/_assets/enable.png)

  3. Сохраните изменения.

- **Настройте нужные параметры**

  1. Откройте вкладку **POSTBACK CONFIG**.

     ![Branch — POSTBACK CONFIG](https://cdn-viewer.diplodoc.com/docs-assets/support-direct-pro/rev/r20128576/ru/_assets/postback.png)

  2. Включите отправку событий **INSTALL**.
  3. Включите отправку событий InApp. Для каждого укажите имя события в Директе. Для этого в поле **POSTBACK URL** замените параметр `&event=${(name)!}` на `&event=EVENT_NAME`, где `EVENT_NAME` — одно из значений:

     - `ACHIEVED_LEVEL` — достижение цели (Completed goal);
     - `ADDED_TO_CART` — добавление в корзину (Added to cart);
     - `ADDED_PAYMENT_INFO` — добавление информации об оплате (Added payment info);
     - `ADDED_TO_WISHLIST` — добавление в список желаний (Added to wish list);
     - `APP_LAUNCHED` — запуск приложения (Launched app);
     - `COMPLETED_REGISTRATION` — завершение регистрации (Completed registration);
     - `COMPLETED_TUTORIAL` — завершение обучения (Completed tutorial);
     - `EVENT_1` — пользовательское событие 1 (Custom event 1);
     - `EVENT_2` — пользовательское событие 2 (Custom event 2);
     - `EVENT_3` — пользовательское событие 3 (Custom event 3);
     - `INITIATED_CHECKOUT` — инициирование заказа (Initiated order);
     - `INSTALL_APP` — установка приложения (AppInstall);
     - `PURCHASED` — совершение покупки (Made a purchase);
     - `RATED` — проставление рейтинга (Submitted rating);
     - `SEARCHED` — выполнение поиска (Performed search);
     - `SHARED` — пользователь поделился (User shared);
     - `SPENT_CREDITS` — потрачены кредиты (Spent credits);
     - `SPENT_TIME_IN_APP` — проведение времени в приложении (Time spent in app);
     - `UNLOCKED_ACHIEVEMENT` — достижение разблокировано (Achievement unlocked);
     - `VIEWED_CONTENT` — просмотр контента (Content viewed).

  4. Для верификации приложения для всех событий в поле **POSTBACK URL** добавьте параметр `&ya-app-id=APP-ID`, где `APP-ID` — идентификатор приложения в Директе.
  5. Для всех событий кампаний, которые используют электронную коммерцию, в поле **POSTBACK URL** добавьте параметр `&ecom_feed_counter_id=ECOM-ID`.
  6. Включите отправку всех событий. Для этого поставьте галочку в поле **ALL EVENTS**.

     ![Branch — ALL EVENTS](https://cdn-viewer.diplodoc.com/docs-assets/support-direct-pro/rev/r20128576/ru/_assets/all-events.png)

     Это позволит:

     - не таргетироваться на уже установивших приложение пользователей;
     - запускать [ретаргетинговые кампании](ru/products-mobile-apps-ads/retargeting-mobile-apps);

### Продвижение на Connected TV и десктопах

Для запуска кросс‑платформенного продвижения приложений на Connected TV и десктопах вам понадобятся специализированные трекинговые ссылки. Настройка доступна в трекерах [AppMetrica](https://appmetrica.yandex.ru/docs/ru/mobile-tracking/ctv-desktop-attribution), [AppsFlyer](https://support.appsflyer.com/hc/ru/articles/8528418936977-Просмотр-через-CTV-в-мобильных-приложениях-для-рекламных-сетей), [Adjust](https://help.adjust.com/ru/article/measure-connected-tv-mobile-campaigns).

#### AppMetrica

**Создайте трекер**

1. В интерфейсе [AppMetrica](https://appmetrika.yandex.ru) перейдите в раздел **Трекинг** → **Создать трекер**.
2. Укажите название трекера.
3. Выберите партнера — **Yandex.Direct**.

   Если вам необходимо выбрать другого партнера для оптимизации кампании в Яндекс Директ, напишите в отдел клиентского сервиса, приложив ID трекера.

**Включите кросс‑платформенную атрибуцию в настройках трекера**

1. В разделе **Трекинг** выберите нужный трекер.
2. На вкладке **Настройка атрибуции** включите опцию **Атрибуция по CTV/Desktop** и укажите окно атрибуции для CTV и/или Desktop-устройств.
3. Полученную **Impression url** используйте в кампаниях.

Подробнее о [настройке](https://appmetrica.yandex.ru/docs/ru/mobile-tracking/ctv-desktop-attribution#setup) в справке AppMetrica.

#### AppsFlyer

**Включите кросс‑платформенную атрибуцию в настройках трекера**

1. В разделе **Engagement** в левом боковом меню нажмите **Onelink management**.
2. В правом верхнем углу нажмите значок ![Меню](https://cdn-viewer.diplodoc.com/docs-assets/support-direct-pro/rev/r20128576/ru/_assets/kebab.png) и выберите **New Onelink template**.
3. В шаблоне включите опцию **Cross-platform attribution is enabled** для поддержки кросс‑платформенной атрибуции.

**Подготовьте трекинговую ссылку**

1. В меню выберите раздел **Collaborate** → **Partner Marketplace**.
2. В списке партнеров выберите Yandex Direct.
3. Для настройки трекинговой ссылки нажмите кнопку **Set up integration/Manage integration**.
4. Перейдите на вкладку **Attribution link**. Выберите **OneLink** и заранее созданный шаблон с поддержкой кросс‑платформенной атрибуции.
5. В блоке **View-through attribution** включите опцию **Cross-platform attribution** и установите окно атрибуции. Сохраните настройки.
6. Скопируйте готовую ссылку.

> **Важно**
>
> Убедитесь, что используются обязательные параметры:
>
> - `?pid=yandexdirect_int` — название площадки;
> - `&af_xplatform=true` — включение кросс‑платформенной атрибуции;
> - `&af_xplatform_vt_lookback=24h` — окно кросс‑платформенной атрибуции;
> - `&clickid={logid}` — уникальный номер показа.

**Равный приоритет для кликов и показов**

Чтобы AppsFlyer рассматривал показы (например, для CTV и консолей) наравне с кликами, в настройках кросс‑платформенной атрибуции включите опцию **Cross-platform equal attribution priority**.

Равенство будет действовать в окне атрибуции от 1 до 6 часов.

#### Adjust

1. Если вы впервые настраиваете интеграцию, на главной странице в блоке **Campaign Lab** нажмите **New Partner**.

   Если вы уже настраивали интеграцию, перейдите в раздел **New partner/Partners**.

2. В списке партнеров выберите Yandex Direct и перейдите в окно настроек.

4. В разделе **Link structure** выберите **Cross-device link**.
5. Установите окна атрибуции.

   Для эффективного отслеживания ретаргетинга установите окна неактивности и окна реатрибуции.

6. Сохраните изменения.
7. В разделе **Partners** скопируйте готовую кросс‑платформенную ссылку для вашего приложения.

   > **Важно**
   >
   > Убедитесь, что используются обязательные параметры:
   >
   > - `&ya_click_id={logid}` — уникальный номер показа;
   > - `&ip_address={client_ip}` — IP адрес пользователя;
   > - `&cross_device_tracking=1` — параметр для включения кросс‑платформенной атрибуции;
   > - `&device_type=android` — для продвижения Android-приложения;
   > - `&device_type=ios` — для продвижения iOS-приложения.

## Диплинк

Диплинк (от англ. *deep link*) — это обязательный параметр трекинговой системы, который добавляется в кампанию на ретаргетинг. Он нужен, чтобы пользователь при клике по объявлению переходил в определенный раздел приложения.

Добавьте при создании [кампаний на ретаргетинг](ru/products-mobile-apps-ads/retargeting-mobile-apps):

- в [Мастере кампаний](ru/products-mobile-apps-ads/create) в поле **Трекинговая ссылка**;
- в [Единой перфоманс-кампании](ru/products-mobile-apps-ads/upc/about) с целью **Ретаргетинг** в поле **Диплинк** при создании объявления.

Для подготовки диплинка обратитесь к разработчику приложения.

<a id="delayed-deeplink"></a>

## Отложенный диплинк

Отложенный диплинк (от англ. *deep link*) — это дополнительный параметр для кампаний с целью **Привлечение новых пользователей**. Он нужен, чтобы пользователь после установки переходил в определенный раздел приложения.

Добавьте:

- в [Мастере кампаний](ru/products-mobile-apps-ads/create) в поле **Трекинговая ссылка**. Для кампаний, запущенных на новую аудиторию;
- в [Единой перфоманс-кампании](ru/products-mobile-apps-ads/upc/about) в поле **Отложенный диплинк** при создании объявления. Доступно только для кампаний с целью **Привлечение новых пользователей**.

Для подготовки диплинка обратитесь к разработчику приложения.

## Кастомные страницы приложения

Доступны для [Google Play](https://play.google.com/store/) и [App Store](https://www.apple.com/app-store/). По клику на объявление новые пользователи обычно переходят на основную страницу приложения в магазине. Но вы можете направить их на кастомную страницу, для этого нужно вставить полную ссылку на нее. Это позволит прорабатывать разные сценарии действий пользователя и подсвечивать преимущества приложения для разных сегментов аудитории.

Добавьте:

- в [Мастере кампаний](ru/products-mobile-apps-ads/create) ссылку на кастомную страницу в параметр редиректа;
- в [Единой перфоманс-кампании](ru/products-mobile-apps-ads/upc/about) в поле **Кастомная страница приложения** при создании объявления. Доступно только для кампаний с целью **Привлечение новых пользователей**.

Для подготовки кастомной страницы приложения воспользуйтесь инструкцией магазина приложений:

- [Google Play](https://support.google.com/googleplay/android-developer/answer/9867158?hl=ru);
- [App Store](https://developer.apple.com/app-store/custom-product-pages/) (на английском языке).
