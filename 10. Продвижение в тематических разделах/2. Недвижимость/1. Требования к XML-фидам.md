# Требования к XML-фидам

Для запуска показов объявлений в тематических разделах поиска для индустрии недвижимости необходимо подготовить два отдельных файла:

1. Фид с информацией о рекламных кампаниях.
2. Фид с информацией об объектах и условиях сделки.

## 1. Требования к информации о рекламных кампаниях

### Общий формат данных

- Кодировка файла: UTF‑8.
- Документ должен начинаться со стандартного XML‑заголовка — с нулевого символа первой строки.
- Для указания пространства имен используйте атрибут `xmlns`.

Пример:

```
xmlns="http://webmaster.yandex.ru/schemas/feed/metarealty/campaigns/2024-12"
```

- Корневой элемент — `campaign-feed` содержит опциональное поле `generation-date` (дата и время создания фида в формате `YYYY-MM-DDThh:mm:ss+00:00`), а также опциональные элементы `campaign`, каждый из которых описывает одну рекламную кампанию.

Если в фиде присутствует `generation-date`, это поле должно предшествовать всем элементам `campaign`.

- Передача фида с нулевым количеством элементов `campaign` остановит все рекламные кампании без удаления фида.

Пример структуры фида:

```xml
<?xml version="1.0" encoding="utf-8"?>
<campaign-feed xmlns="http://webmaster.yandex.ru/schemas/feed/metarealty/campaigns/2024-12">
  <generation-date>2024-12-25T12:34:56+03:00</generation-date>
  <campaign>
    ...
  </campaign>
  ...
</campaign-feed>
```

Чтобы выключить рекламную кампанию:

1. Удалите фид со ставками. При этом ваши рекламные кампании перейдут в ручное управление.
2. Выключите все ЖК на странице **Кампании**.

### Элементы, входящие в `<campaign>`

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| `newbuilding-id` | Да | Идентификатор ЖК |
| `cost-per-call` | Да | Ставка за звонок |
| `phone` | Да | Номер телефона |
| `sip` | Нет | SIP‑адрес |
| `work-schedule` | Нет | Часы работы |

**Идентификатор ЖК**

- Элемент `newbuilding-id` содержит идентификатор жилого комплекса в базе Яндекса и определяет, для какого именно ЖК включается размещение.
- Отсутствие элемента или недопустимое значение препятствует созданию кампании.
- Если в фиде несколько элементов `campaign` для одного ЖК, один из них будет выбран для создания или обновления кампании случайным образом.

**Ставка за звонок**

- Элемент `cost-per-call` целочисленный, содержит стоимость целевого звонка по ЖК в копейках.
- Значение ставки за звонок, передаваемое в фиде, должно соответствовать минимальной ставке за целевой звонок.
- Если передано значение ниже минимального для региона и класса жилья, оно будет поднято до ближайшего допустимого.
- При обновлении кампании невалидное значение будет проигнорировано.

**Номер телефона**

- Элемент `phone` содержит номер телефона для звонков по ЖК в формате `+7XXXXXXXXXX`.
- Номер должен начинаться с `+7` (номера формата `8‑800...` использовать нельзя).
- Отсутствие элемента или неверный формат препятствует созданию кампании.
- При обновлении невалидный номер будет проигнорирован.

**SIP-адрес**

- Элемент `sip` содержит SIP-адрес для связи по ЖК в формате `sip:user@host.com`
- Отсутствие элемента или неверный формат передаваемого SIP-адреса не влияют на создание или обновление кампании.

**Часы работы**

```xml
<work-schedule>
  <timezone>Europe/Moscow</timezone>
  <pattern day-from="1" day-to="5" start-time="09:00" end-time="21:00"/>
  <pattern day-from="6" day-to="7" start-time="10:00" end-time="19:00"/>
</work-schedule>
```

- Элемент `work-schedule` содержит описание рабочего времени отдела продаж, которое учитывается при выставлении счетов за неотвеченные вызовы.
- Элемент включает обязательное текстовое поле `timezone` (часовой пояс в формате континент/город либо оффсет) и одно или несколько полей `pattern` с описанием формата работы.
- Элементы `pattern` не должны пересекаться — каждому дню недели должен соответствовать не более чем один промежуток рабочего времени.
- Отсутствие элемента или неверный формат не влияют на создание или обновление кампании.
- Если невозможно сформировать расписание из переданных данных, будет использовано расписание по умолчанию: московское время, Пн–Вс 09:00–21:00.
- Элемент `pattern` содержит описание часов работы. Элемент включает обязательные атрибуты:
  - `day-from` и `day-to` (диапазон дней недели);
  - `start-time` и `end-time` (часы работы).
- Дни недели можно задать:
  - численно (от 1 — понедельник до 7 — воскресенье);
  - текстом на английском или русском языках (`Monday/Mon`, `Понедельник/Пн` и т. д.).
- Время должно быть передано в формате ISO 8601 для времени дня (`hh:mm` или `hh:mm:ss`).

## 2. Требования к информации об объектах и условиях сделки

### Общий формат данных

- Кодировка файла: UTF‑8.
- Документ должен начинаться со стандартного XML‑заголовка — с нулевого символа первой строки.
- Для указания пространства имен используйте атрибут `xmlns`.

Пример:

```
xmlns="http://webmaster.yandex.ru/schemas/feed/metarealty/2024-12"
```

- Корневой элемент — `realty-feed`. Он содержит опциональное поле `generation-date` (дата и время создания фида в формате `YYYY-MM-DDThh:mm:ss+00:00`) и обязательные элементы `offer`, каждый из которых описывает один оффер.

Если в фиде присутствует `generation-date`, это поле должно предшествовать всем элементам `offer`.

Пример структуры фида:

```xml
<?xml version="1.0" encoding="utf-8"?>
<realty-feed xmlns="http://webmaster.yandex.ru/schemas/feed/metarealty/2024-12">
  <generation-date>2024-12-25T12:34:56+03:00</generation-date>
  <offer internal-id="А1245">
    ...
  </offer>
  ...
</realty-feed>
```

### Общая информация об объявлении

Под объявлением (единица элемента `offer`) понимается информация о конкретной квартире в новостройке.

У элемента `offer` есть обязательный атрибут `internal-id`, который должен быть уникальным для каждого объявления. Атрибут `internal-id` может состоять из цифр, латинских и кириллических букв, знаков. Рекомендуется заполнять его номером из вашей базы данных.

> **Внимание**
> Элементы со строго ограниченным набором значений могут содержать только перечисленные варианты. Использование других значений приведет к ошибке обработки фида.

### Элементы, входящие в `<offer>`

**Основные элементы объявления**

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| `type` | Да | Тип сделки. Строго ограниченные значения: «продажа». |
| `category` | Да | Категория объекта. Возможные значения: «квартира»/«flat». |
| `url` | Нет | URL страницы с объявлением на вашем сайте (не показывается в объявлении). |
| `location` | Да | Внутри передаются элементы о расположении объекта: `address`, `latitude`, `longitude`, `apartment`. |
| `address` | Да | Адрес объекта (улица и номер здания). Передается внутри `location`. |
| `apartment` | Нет | Номер квартиры. Передается внутри `location`. |
| `latitude` | Нет | Географическая широта. Передается внутри `location`. |
| `longitude` | Нет | Географическая долгота. Передается внутри `location`. |

**Информация о продавце**

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| `category` | Нет | Тип продавца. Строго ограниченные значения: «застройщик»/«developer». |

**Информация об условиях сделки**

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| `price` | Да | Набор элементов, описывающих цену объекта. |
| `value` | Да | Цена (вводится без пробелов). Пример: `<value>30000.55</value>`. |
| `currency` | Да | Валюта. Возможные значения: «RUR» или «RUB» (российский рубль). |
| `unit` | Нет | Единица площади помещения (нужна, только если цена указана за единицу площади). |

**Дополнительная информация об условиях сделки**

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| `deal-status` | Да | Тип сделки. Возможные значения: «214фз», «продажа от застройщика»/«primary», «primary sale of secondary»/«первичная продажа вторички». |

**Информация об объекте**

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| `area` | Да | Общая площадь. |
| `room-space` | Нет | Площадь комнаты. |
| `living-space` | Нет | Жилая площадь. |
| `kitchen-space` | Нет | Площадь кухни. |
| `image` | Да | Фотография. Может быть несколько тегов. Фотографии планировок следует передавать первым тегом `image`. |
| `description` | Нет | Дополнительная информация. Ограничение — 10 000 знаков. |

**Описание жилого помещения**

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| `rooms` | Нет | Общее количество комнат в квартире. |
| `floor` | Нет | Этажи, на которых находится объект. |
| `apartment` | Нет | Апартаменты. Значения: «да»/«нет», «true»/«false», «1»/«0», «+»/«-». |
| `studio` | Нет | Студия. Значения: «да»/«нет», «true»/«false», «1»/«0», «+»/«-». |
| `balcony` | Нет | Тип балкона. |
| `decoration-type` | Нет | Тип отделки. |
| `bathroom-unit` | Нет | Тип санузла. |

**Описание здания**

| Элемент | Обязательность | Описание |
|---------|---------------|----------|
| `floors-total` | Нет | Общее количество этажей в доме. |
| `building-name` | Нет | Название жилого комплекса. |
| `yandex-building-id` | Да | Идентификатор жилого комплекса в базе Яндекса. |
| `yandex-house-id` | Да | Идентификатор корпуса жилого комплекса в базе Яндекса. |
| `building-type` | Нет | Тип дома. |
| `heating-type` | Нет | Тип отопления. |
| `built-year` | Нет | Год сдачи. |
| `ready-quarter` | Нет | Квартал сдачи дома. Значения: «1», «2», «3» или «4». |
| `ceiling-height` | Нет | Высота потолков в метрах. |
| `guarded-building` | Нет | Закрытая территория. |
| `access-control-system` | Нет | Наличие пропускной системы. |
| `twenty-four-seven` | Нет | Круглосуточный доступ. |
| `eating-facilities` | Нет | Общепит в здании. |
| `lift` | Нет | Лифт. |
| `rubbish-chute` | Нет | Мусоропровод. |
| `parking` | Нет | Наличие охраняемой парковки. |
| `parking-guest` | Нет | Наличие гостевых парковочных мест. |
| `flat-alarm` | Нет | Наличие сигнализации в квартире. |
| `security` | Нет | Наличие охраны. |
| `is-elite` | Нет | Элитная недвижимость. |
| `building-state` | Нет | Стадия строительства дома. Значения: `built`, `hand-over`, `unfinished`, `construction-suspended`, `in-project`. |
