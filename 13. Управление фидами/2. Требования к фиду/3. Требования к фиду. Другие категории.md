# Требования к фиду. Другие категории

## Фид «Отели и аренда жилья» Google Рекламы

Фид «Отели и аренда жилья» Google Рекламы должен быть в формате CSV. Первая строка содержит названия столбцов, а следующие строки ― сами данные. Содержимое столбцов отделяется друг от друга запятой. Данные в фиде должны быть в кодировке UTF-8.

**Пример:**

[Скачать пример фида для отелей в формате CSV](https://yastatic.net/s3/doc-binary/src/support/direct/ru/hotels_feed_example.csv)

**Элементы, входящие в CSV**

В таблице ниже описаны элементы фида, которые используются для создания товарных объявлений и объявлений для страниц каталога в Единой перфоманс-кампании.

| **Элемент** | **Описание** |
| --- | --- |
| `Property ID` | Идентификатор отеля.<br><br>Обязательный элемент. |
| `Category` | Тип размещения. Например:<br><br>- «отель»<br>- «апартаменты»<br>- «хостел» |
| `Property name` | Название отеля.<br><br>Обязательный элемент. |
| `Final URL` | URL страницы предложения для продвижения сайта или трекинговая ссылка для продвижения мобильного приложения. Если вы продвигаете мобильное предложение, рекомендуем добавить отложенные диплинки.<br><br>Обязательный элемент. |
| `Destination name` | Местоположение отеля (не более 25 символов).<br><br>Обязательный элемент. |
| `Image URL` | Ссылка на изображение. Рекомендуемый размер изображения — минимум 500 пикселей по одной из сторон.<br><br>Обязательный элемент для показов в РСЯ. |
| `Price` | Цена предложения. Число и код валюты (`RUB`, `USD`, `UAH`, `KZT`). Используйте точку (.) в качестве десятичного разделителя.<br><br>Рекомендуемый элемент. |

| `Max score` | Максимально возможная оценка, целое число. По умолчанию 5. |
| `Facilities` | Услуги отеля. Укажите значения через точку с запятой: `бар;бассейн;бесплатный Wi-Fi`. |

Длина одного элемента — до 175 символов.<br><br>Кастомные элементы фида могут использоваться только для создания фильтров в Единой перфоманс-кампании. На генерацию креативов они не влияют. Фильтры используются для сегментирования оферов по специфичным параметрам, для которых не предусмотрены стандартные фильтры. Например, можно задать фильтр по принадлежности отеля к конкретной акции. |
| `custom_score` | Кастомный элемент. Цифровое значение&nbsp;— ноль или целое положительное число.<br><br><br><br>Кастомные элементы фида могут использоваться только для создания фильтров в Единой перфоманс-кампании. На генерацию креативов они не влияют. Фильтры используются для сегментирования оферов по специфичным параметрам, для которых не предусмотрены стандартные фильтры. Например, можно задать фильтр по маржинальности предложения. |

> **Примечание**
>
> Значения, содержащие запятую, нужно заключить в кавычки (""). Например, "Москва, центр".

## Фид Авто.ру {#autos}

Фид Авто.ру должен быть в формате XML. Данные в фиде должны быть в кодировке UTF-8.

Описание формата XML легковых авто в [Справке Авто.ру](https://yandex.ru/support/autoru-legal/requirements-cars.html).

Список предложений бизнеса содержится в элементе `<cars>`. Каждый автомобиль описывается отдельным элементом `<car>`.

**Пример:**

#### Для нового автомобиля

```xml
    <?xml version="1.0"?>
    <data>
        <cars>
            <car>
                <mark_id>Hyundai</mark_id>
                <folder_id>Sonata</folder_id>
                <modification_id>2.0 AT (150 л.с.)</modification_id>
                <url>http://www.auto.ru/1</url>
                <images>http://picture.auto.ru/1.jpg</images>
                <body_type>Седан</body_type>
                <color>серый</color>
                <availability>в наличии</availability>
                <custom>растаможен</custom>
                <year>2022</year>
                <price>2575000</price>
                <currency>RUR</currency>
                <vin>XWBCA41ZXDK259205</vin>
                <owners_number>Не было владельцев</owners_number>
              </car>
          </cars>
    </data>
  ```

#### Для подержанного автомобиля

```xml
    <?xml version="1.0"?>
    <data>
        <cars>
            <car>
                <mark_id>Nissan</mark_id>
                <folder_id>Juke, I Рестайлинг</folder_id>
                <modification_id>1.6 CVT (117 л.с.)</modification_id>
                <url>http://www.auto.ru/2</url>
                <body_type>Внедорожник 5 дв.</body_type>
                <color>белый</color>
                <availability>в наличии</availability>
                <custom>растаможен</custom>
                <state>среднее</state>
                <owners_number>Один владелец</owners_number>
                <run>60000</run>
                <year>2017</year>
                <price>550000</price>
                <currency>RUR</currency>
                <vin>XWBCA41ZXDK259107</vin>
                <images>
                    <image>http://price.site.ru/files/image2.jpg</image>
                    <image>http://price.site.ru/files/image3.jpg</image>
                </images>
              </car>
          </cars>
    </data>
  ```

В фиде можно передать информацию о каталогах автомобилей. В общей структуре файла элемент `<collections>` идет после элемента `<cars>`.

### Элементы, входящие в `<collections>`

| **Элемент** | **Описание** | **Обязательный элемент** |
| --- | --- | --- |
| `collection id` | Уникальный ID каталога, не более 60 символов. Может быть числовым, буквенным, буквенно-числовым.<br><br><br><br>Идентификаторы для каждого каталога должны быть уникальны. | да |
| `url` | URL страницы каталога.<br><br><br><br>Максимальная длина ссылки&nbsp;— 2048 символов. Допускаются кириллические ссылки. (Кириллические ссылки рекомендуем преобразовать с помощью [Punycode](https://www.punycoder.com/).) URL‑адрес формируется на основе стандарта [RFC 3986](https://datatracker.ietf.org/doc/html/rfc3986).<br><br><br><br>В URL обязательно указать протокол `http` или `https`.<br><br><br><br>  #### Как кодировать символы в URL?<br><br>\| **Символ в тексте** \| **Код для YML-файла** \|<br><br>\| --- \| --- \|<br><br>\| `"` \| `&quot;` \|<br><br>\| `&` \| `&amp;` \|<br><br>\| `>` \| `&gt;` \|<br><br>\| `<` \| `&lt;` \|<br><br>\| `'` \| `&apos;` \|<br><br><br><br> URL не должен содержать пробелов. Их нужно преобразовать в `%20`.<br><br><br><br> | да |
| `picture` | Ссылка на изображение.<br><br><br><br>Можно добавить несколько ссылок на изображения, каждую в собственном элементе picture.<br><br><br><br>#### Как изменить картинку в объявлении?<br><br>Чтобы робот Яндекса обновил изображение каталога в объявлениях, нужно изменить ссылку на него в фиде​. Например:<br><br><br><br>- ссылка на текущее изображение&nbsp;— `<picture>http://auto.ru/newavto/00016.jpg</picture>`<br><br>- ссылка на новое изображение&nbsp;— `<picture>http://auto.ru/newavto/00016-new.jpg</picture>`<br><br><br><br> | для показов в РСЯ |
| `name` | Название каталога. | да |
| `description` | Описание каталога. | нет |

### Пример для страниц каталога

```xml
<?xml version="1.0" encoding="utf-8"?>
<data>
  <cars>
    <car></car>
    <car></car>
  </cars>
  <collections>
    <collection id="1">
      <url>https://auto.ru/new</url>
      <name>Новые автомобили в наличии</name>
      <description>Купить новые автомобили у официального дилера</description>
      <picture>http://auto.ru/newavto/00016.jpg</picture>
    </collection>
  </collections>
</data>
```

## Фид Яндекс Недвижимости {#real-estate}

Фид Яндекс Недвижимости должен быть в формате YRL (Yandex Realty Language). Подробнее о формате YRL в [Справке Яндекс Недвижимости](https://yandex.ru/support/realty/partners/requirements.html).

Каждое товарное предложение описывается отдельным элементом `<offer>`.

В одном фиде можно передавать информацию о разных ЖК разных застройщиков. Для этого укажите в каждом `<offer>` правовую информацию о застройщике в элементе `<sales-agent>`.

**Пример:**

```xml
<offer internal-id="15782295">
  <type>продажа</type>
  <property-type>жилая</property-type>
  <category>квартира</category>
  <url>http://www.developer.ru/search18</url>
  <creation-date>2015-04-02T19:00:06+03:00</creation-date>
  <location>
    <locality-name>Санкт-Петербург</locality-name>
    <sub-locality-name>о. Васильевский</sub-locality-name>
    <address>18-я линия В.О., 32</address>
    <metro>
      <name>Василеостровская</name>
      <time-on-foot>10</time-on-foot>
      <time-on-transport>5</time-on-transport>
   </metro>
  </location>
  <deal-status>первичная продажа</deal-status>
  <price>
    <value>4780000</value>
    <currency>RUR</currency>
  </price>
  <sales-agent>
    <organization>ЗАО "Застройщик"</organization>
  </sales-agent>
  <rooms>2</rooms>
  <floor>13</floor>
  <floors-total>15</floors-total>
  <area>
    <value>63.00</value>
    <unit>кв. м</unit>
  </area>
  <building-name>Северная фантазия</building-name>
  <building-state>unfinished</building-state>
  <ready-quarter>3</ready-quarter>
  <image>http://www.developer.ru/images/plans/000001289.jpg</image>
  <image>http://www.developer.ru/images/plans/000001290.jpg</image>
  <description>Продается 2 к. кв., 13 этаж, 15 минут на машине до метро «Василеостровская».
  Дом комфорт-класса с продуманными планировочными решениями и широким выбором квартир.
  Внутренний двор «Северной фантазии» выполнен по эксклюзивному дизайн-проекту.
  В районе постройки нового ЖК развита инфраструктура: школы и детские сады, больница, аптеки магазины, кафе и спортивные центры.
  Доступны разные условия ипотеки, скидки и зачет жилья.</description>
</offer>
```

### Пример для страниц каталога

```xml
<?xml version="1.0" encoding="utf-8"?>
<realty-feed>
  <offer>
  </offer>
  <collections>
    <collection id="1">
      <url>https://developer.ru/#/macrocatalog/complexes/1111111</url>
      <name>Двухкомнатные квартиры на Васильевском острове</name>
      <description>Двухкомнатные квартиры на Васильевском острове с отделкой от застройщика и современной планировкой</description>

    </collection>
  </collections>
</realty-feed>
```

Фид «Путешествия» Google Рекламы должен быть в формате CSV. Первая строка содержит названия столбцов, а следующие строки — сами данные. Содержимое столбцов отделяется друг от друга запятой. Данные в фиде должны быть в кодировке UTF-8.

**Пример**

[Скачать пример фида «Путешествия» Google Рекламы в формате CSV](https://yastatic.net/s3/doc-binary/src/support/direct/ru/files/google_travel_feed_example.csv)
