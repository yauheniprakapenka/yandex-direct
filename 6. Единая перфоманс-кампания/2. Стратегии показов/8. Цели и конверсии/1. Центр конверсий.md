Загружайте данные о конверсиях и учитывайте их при настройке кампаний. Соберите все данные о конверсиях из всех источников и анализируйте картину целиком:

- какие конверсии из офлайна или по звонкам принесли объявления в Директе;
- сколько стоили конверсии из Директа.

Все это доступно в Центре конверсий — едином окне для работы с конверсиями из офлайна, по звонкам и из онлайна. В нем можно:

- загружать конверсии из офлайна и по звонкам альтернативными способами отправки данных. Это дополнит сведения Метрики о конверсиях. Подробнее о загрузке [офлайн-конверсии](ru/strategies/offline-conversions) и [конверсий по звонкам](ru/strategies/call-conversions).
- отслеживать статистику по конверсиям, которые загружены через альтернативные способы, по API или через коннекторы.

Чтобы перейти в центр, в меню выберите **Конверсии**.

## Загрузка данных о конверсиях

Данные о конверсиях можно выгрузить из CRM-базы в CSV-файл и добавить через альтернативные способы загрузки в Центр конверсий: с FTP или SFTP-сервера, по ссылке на HTTP или HTTPS или через Google Sheets. Из Центра конверсий данные попадут в Метрику, где по ним будут сформированы конверсии и цели.

Статистика и оптимизация на цели будет доступна для всех логинов Директа, у которых есть доступ к счетчику Метрики, в который загружены данные.

Загрузка состоит из трех этапов:

1. Сформируйте файл с данными по установленному формату.
2. Загрузите файл в источник.
3. Добавьте ссылку на источник.

### Этап 1. Подготовка файла

Сформируйте файл в формате CSV. Используйте точку с запятой, чтобы разделить колонки.

**Обязательные поля:**

| Название | Описание |
|---|---|
| `create_date_time` | Дата и время создания заказа в часовом поясе счетчика. Значение нельзя менять. Указывайте дату и время. Если не храните время — отправляйте только дату (автоматически припишется 23:59:59). |
| `client_ids` | Список ClientID Яндекс Метрики. |
| `emails` | Список адресов электронной почты клиента. Строка латиницей, включающая символ @ и имя домена. Прописные буквы использовать нельзя. |
| `phones` | Список телефонных номеров клиента. Числовая строка с кодом страны. Без пробелов и доп. символов. Пример: `79995551111`. |
| `emails_md5` | Список email, хешированных в md5. |
| `phones_md5` | Список телефонов, хешированных в md5. |

Обязательно хотя бы одно из полей идентификаторов (`client_ids`, `emails`, `phones`, `emails_md5`, `phones_md5`).

**Необязательные поля:**

| Название | Описание |
|---|---|
| `id` | Идентификатор заказа в CRM. Позволяет обновлять статус заказа и перепривязывать заказ между клиентами. |
| `client_uniq_id` | Идентификатор клиента в CRM, связанного с заказом. Позволяет склеивать будущие заказы с прошлыми. |
| `order_status` | Тип статуса заказа: `PAID` (оплачен), `IN_PROGRESS` (создан, но не оплачен), `CANCELLED` (отменен), `SPAM`. Можно использовать идентификатор JavaScript-цели. |
| `revenue` | Доход. Суммарная стоимость заказа. Десятичная дробь (разделитель — точка). |
| `cost` | Себестоимость. Десятичная дробь. |

Конверсии со статусом `IN_PROGRESS` добавляются в цель «CRM: Заказ создан». Со статусом `PAID` — в цель «CRM: Заказ оплачен». Если отправляете заказы только со статусом `PAID`, конверсии добавляются в обе цели.

[Пример CSV-файла](https://yastatic.net/s3/doc-binary/src/support/direct/ru/simple_orders_conv.csv)

### Этап 2. Загрузка файла в источник

Настройте загрузку данных в любой из подходящих источников: FTP-сервер, SFTP-сервер, ссылка HTTP/HTTPS, Google Sheets.

Важно соблюдать 2 правила:

1. Добавляйте в файл новые конверсии в реальном времени или не реже одного раза в день.
2. Поддерживайте в файле актуальность конверсий за последние три дня.

### Этап 3. Добавление ссылки на источник

1. Перейдите на страницу **Конверсии** и нажмите **Добавить источник конверсий**.
2. Выберите подходящий источник для загрузки: файл по ссылке, файл по FTP, файл по SFTP или Google Sheets.
3. Укажите название источника, добавьте необходимые данные для подключения (ссылку, логин/пароль, порт, хост или ссылку на Google Sheets).
4. Выберите счетчик Метрики для загрузки данных.
5. При необходимости укажите ценность конверсий.

Для Google Sheets предоставьте доступ с уровнем «Просмотр» для аккаунта `import@yandex-direct-conversions.iam.gserviceaccount.com`.

Добавить источник достаточно на одном логине Директа. После загрузки данных в Метрике будут созданы цели, которые можно использовать на всех логинах с доступом к счетчику.

## Использование идентификаторов JavaScript-целей в качестве статуса заказа

В колонке `order_status` помимо фиксированных статусов можно использовать идентификатор JavaScript-цели.

Для этого создайте отдельную цель в Метрике: **Метрика** → **Цели** → **Добавить цель** → тип **Java-Script-событие** → задайте идентификатор, который не будет срабатывать от действий на сайте.

В файле с конверсиями в колонке `order_status` укажите идентификатор цели.

Статусы заказа чувствительны к регистру.

## Окно атрибуции конверсии к визитам

Окно атрибуции, в течение которого можно привязать конверсию к визиту, составляет **21 день**. Визит не дополнится информацией о конверсии, если она совершена позже 21 дня с последнего визита.

Когда визит был дополнен информацией о конверсии, у вас есть еще **90 дней** с момента окончания 21-дневного окна на то, чтобы скорректировать или дополнить эту конверсию.

## Как проверить, что все работает

1. Убедитесь, что в созданной выгрузке есть оба вида заказов (онлайн и офлайн).
2. Убедитесь, что выгрузка обновляется раз в сутки или чаще.
3. После первой выгрузки в течение 24 часов зайдите в Метрику и проверьте, что в **Конверсиях** появились цели «CRM: Заказ создан» и «CRM: Заказ оплачен» (или статистика по JavaScript-цели).
4. Для корректного обновления статусов заказов отправляйте в выгрузке `id` заказа из CRM.

## Ошибки при загрузке

Если при добавлении источника появилась красная плашка, файл не загрузился. Основные ошибки:

- **Ошибка соединения с сервером** — проверьте порт, хост и путь к файлу.
- **Нет доступа к счетчику Метрики** — убедитесь, что логин имеет доступ на редактирование.
- **Ошибка при скачивании файла** — проверьте настройки, попробуйте скачать файл самостоятельно.
- **Неверные данные авторизации** — проверьте логин и пароль к серверу.
- **Неправильный разделитель** — используйте точку с запятой.
- **Нет валидных данных** — проверьте название колонок, формат даты, телефонов и email.
- **Недостаточно прав для Google Sheets** — предоставьте доступ для аккаунта `import@yandex-direct-conversions.iam.gserviceaccount.com`.
- **Размер файла слишком большой** — максимум 300 Мбайт.
- **Неверная кодировка** — поддерживается только UTF-8.
- **Более одного листа в таблице** — перенесите данные на один лист.

## Ошибки в файле

| Ошибка | Решение |
|---|---|
| `Invalid date format` | Скорректируйте формат даты и времени в `create_date_time`. |
| `Empty 'create_date_time' value` | Укажите дату и время. Поле обязательное. |
| `'create_date_time' is older than 113 days` | Удалите конверсии старше 113 дней. |
| `Invalid '<id_type>' identifier` | Укажите идентификатор пользователя в корректном формате. |
| `All columns with identifiers are empty` | Заполните хотя бы одну колонку с идентификаторами. |
| `Invalid '<revenue/cost>' value` | Укажите revenue/cost как положительное число. |
| `Invalid 'order_status' value` | Используйте корректные статусы или идентификатор JavaScript-цели в правильном регистре. |

## Статистика по конверсиям

В Центре конверсий можно отслеживать статистику по достижению целей, созданных в Метрике:

1. На странице **Конверсии** нажмите **Добавить источник конверсий** → **Яндекс Метрика**.
2. Выберите счетчик Метрики, для которого загружали данные.
3. Выберите цели. При необходимости задайте ценность вручную.

В отчете будут данные:

- **Конверсии** — количество целевых визитов по цели из всех источников.
- **Конверсии из Директа** — количество и доля конверсий после перехода по кампаниям из Яндекс Директа.
